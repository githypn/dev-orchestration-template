## 4. VS Code のインストールと初期設定

### VS Code とは？

**Visual Studio Code（VS Code）** は、Microsoft が開発した無料のコードエディタです。
メモ帳のような「テキストを編集するアプリ」ですが、プログラミングに特化した機能がたくさん付いています。
Copilot もこの VS Code の中で動きます。

### インストール

1. https://code.visualstudio.com にアクセス
2. **macOS 用をダウンロード**（Apple Silicon / Intel を自動判別）
3. ダウンロードした `.zip` を展開
4. `Visual Studio Code.app` を **アプリケーション** フォルダにドラッグ
5. Launchpad または Finder から VS Code を起動

### 日本語化（任意）

1. VS Code を起動
2. 左サイドバーの **拡張機能アイコン**（四角が4つ並んだアイコン）をクリック
3. 検索バーに「Japanese」と入力
4. **Japanese Language Pack for Visual Studio Code** をインストール
5. VS Code を再起動

### コマンドラインから VS Code を起動できるようにする

1. VS Code を開く
2. `Cmd + Shift + P` でコマンドパレットを開く
3. 「shell command」と入力
4. **Shell Command: Install 'code' command in PATH** を選択

これで、ターミナルから `code .` でプロジェクトを開けるようになります。

### VS Code の画面構成

VS Code を初めて開くと、以下のような画面が表示されます。

```
┌─────────────────────────────────────────────────┐
│ メニューバー（File, Edit, View ...）             │
├────┬────────────────────────────────────────────┤
│    │                                            │
│ サ │         エディタ領域                        │
│ イ │     （ファイルの中身を表示・編集）          │
│ ド │                                            │
│ バ │                                            │
│ ー │────────────────────────────────────────────│
│    │         ターミナル / パネル領域             │
│    │     （コマンドを入力する場所）              │
├────┴────────────────────────────────────────────┤
│ ステータスバー（ブランチ名、エラー数 etc.）      │
└─────────────────────────────────────────────────┘
```

- **サイドバー**（左）: ファイル一覧、検索、Git、拡張機能などのアイコンが並ぶ
- **エディタ領域**（中央）: ファイルの中身を開いて編集する場所
- **パネル領域**（下）: ターミナル（後述）やエラー一覧が表示される
- **ステータスバー**（最下部）: 今いるブランチ名やエラー数が表示される

## 5. Git のセットアップ

macOS には Git がプリインストールされています。

### 確認

ターミナル（`Terminal.app` または VS Code のターミナル）を開いて：

```bash
git --version
```

バージョンが表示されれば OK です。
「Xcode Command Line Tools をインストールしますか？」と聞かれたら **インストール** をクリック。

### 初期設定

```bash
# あなたの名前（GitHub のユーザー名でも OK）
git config --global user.name "あなたの名前"

# GitHub に登録したメールアドレス
git config --global user.email "your-email@example.com"
```

### GitHub CLI のインストール（推奨）

GitHub の操作をコマンドラインから行えるツールです。

```bash
# Homebrew がインストールされていない場合、先にインストール
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# GitHub CLI をインストール
brew install gh

# GitHub にログイン
gh auth login
```

`gh auth login` では以下を選択：
- **GitHub.com** を選択
- **HTTPS** を選択
- **Login with a web browser** を選択
- ブラウザが開くので、GitHub にログイン

## 6. リポジトリをクローンして開く

### 「クローン」とは？

GitHub 上にあるリポジトリ（プロジェクトフォルダ）を、自分の Mac にダウンロードすることを **クローン** と呼びます。
ただの「ダウンロード」と違い、変更の履歴や GitHub との連携情報もすべてコピーされます。

### 方法 A: ターミナルから（推奨）

```bash
# 作業用フォルダに移動（例: ホームの GitHub フォルダ）
mkdir -p ~/GitHub
cd ~/GitHub

# リポジトリをクローン
git clone https://github.com/githypn/dev-orchestration-template.git

# VS Code で開く
code dev-orchestration-template
```

### 方法 B: VS Code から

1. VS Code を起動
2. `Cmd + Shift + P` →「Git: Clone」と入力
3. リポジトリ URL を入力: `https://github.com/githypn/dev-orchestration-template.git`
4. 保存先フォルダを選択
5. 「開く」をクリック

## 7. 自分の GitHub で使えるようにする

このテンプレートを自分のプロジェクトとして使うには、**テンプレートから自分のリポジトリを作成** します。
GitHub の「Use this template」機能を使えば、ボタン数クリックで完了します。

### 手順（推奨: GitHub Web から作成）

```
① GitHub Web でテンプレートから自分のリポジトリを作る
② 作成されたリポジトリをクローンする
③ VS Code で開いて開発開始
```

#### ① テンプレートから自分のリポジトリを作る

1. https://github.com/githypn/dev-orchestration-template にアクセス
2. 右上の緑の **「Use this template」** ボタンをクリック
3. **「Create a new repository」** を選択
4. 以下を入力：
   - **Repository name**: 好きなプロジェクト名（例: `my-project`）
   - **Description**: プロジェクトの説明（任意）
   - **Public / Private**: Private（非公開）がおすすめ
5. **「Create repository」** をクリック

> これで、テンプレートの中身がすべてコピーされた **あなた自身のリポジトリ** が作成されます。
> 元のテンプレートとは独立しているので、自由に変更できます。

#### ② クローンして VS Code で開く

作成されたリポジトリのページに表示される URL をコピーして：

```bash
# 作業用フォルダに移動
mkdir -p ~/GitHub
cd ~/GitHub

# あなたのリポジトリをクローン（URL はリポジトリページからコピー）
git clone https://github.com/あなたのユーザー名/my-project.git

# VS Code で開く
code my-project
```

#### ③ 確認する

```bash
# 接続先を確認する
git remote -v
```

以下のように、あなたのリポジトリ URL が表示されれば OK です：

```
origin  https://github.com/あなたのユーザー名/my-project.git (fetch)
origin  https://github.com/あなたのユーザー名/my-project.git (push)
```

### ローカルと GitHub を同期する仕組み

開発中は、ローカル（Mac 上）と GitHub（クラウド上）の間でコードを同期します。

```
Mac 上のフォルダ          GitHub 上のリポジトリ
┌──────────────┐         ┌──────────────┐
│ ファイル編集  │  push → │  最新版を保管  │
│ コード作成    │ ← pull  │  履歴を記録    │
└──────────────┘         └──────────────┘
```

| 操作 | コマンド | 意味 |
|------|---------|------|
| **変更を記録する** | `git add .` + `git commit -m "説明"` | ローカルで「この変更を保存」と記録する |
| **GitHub にアップロード** | `git push` | ローカルの変更を GitHub に反映する |
| **GitHub からダウンロード** | `git pull` | GitHub 上の最新変更をローカルに反映する |

> **普段は Copilot が自動で行います。** 手動で操作する必要があるのは、最初のセットアップとマージ承認だけです。

### 別の方法: コマンドラインで作成

GitHub CLI を使ってターミナルから作成することもできます（上級者向け）。

```bash
# テンプレートをクローン
git clone https://github.com/githypn/dev-orchestration-template.git my-project
cd my-project

# 元のテンプレートとの接続を切り、自分のリポジトリとして登録
rm -rf .git
git init
git add .
git commit -m "initial commit from template"

# GitHub 上にリポジトリを作成してプッシュ
gh repo create my-project --private --source=. --push
```

## 9. VS Code のターミナルの使い方

### ターミナルとは？

**ターミナル** は、テキスト（文字）でパソコンに命令を出す画面です。

普段はマウスでアイコンをクリックして操作しますが、ターミナルでは命令文（コマンド）をキーボードで打ち込んで操作します。
プログラミングでは、ファイルの作成、プログラムの実行、Git の操作などに使います。

```
例：
$ ls                     ← 「今のフォルダにあるファイルを見せて」
README.md  docs/  src/   ← パソコンの回答（ファイル一覧が表示される）
```

### VS Code でターミナルを開く方法

VS Code にはターミナルが内蔵されているので、別のアプリを開く必要はありません。

| 方法 | 手順 |
|------|------|
| **キーボード（最速）** | `` Ctrl + ` ``（Ctrl キーを押しながらバッククォート） |
| **メニューから** | メニューバーの **Terminal** → **New Terminal** |
| **コマンドパレット** | `Cmd + Shift + P` →「Terminal: Create New Terminal」 |

画面下部にターミナルが表示されます。

### ターミナルの見方

```
┌──────────────────────────────────────────┐
│ ユーザー名@Mac名 フォルダ名 %            │  ← プロンプト（入力待ちの状態）
│ █                                        │  ← ここにコマンドを入力する
│                                          │
│                                          │
└──────────────────────────────────────────┘
```

- `%` や `$` のマーク → 「コマンドを入力してください」という意味
- コマンドを入力して `Enter` を押すと実行される
- 結果がその下に表示される

### よく使うコマンド

| コマンド | 意味 | 例 |
|---------|------|-----|
| `ls` | フォルダの中身を表示 | `ls docs/` |
| `cd フォルダ名` | フォルダに移動 | `cd docs` |
| `cd ..` | 一つ上のフォルダに戻る | `cd ..` |
| `pwd` | 現在のフォルダの場所を表示 | `pwd` |
| `clear` | 画面をきれいにする | `clear` |

### ターミナルの複数タブ

ターミナル右上の **＋** ボタンで、新しいターミナルを追加できます。
複数のターミナルを切り替えて使えるので、1つで Copilot が作業中でも、別のターミナルで自分の作業ができます。

> **普段はターミナルを直接操作する必要はほとんどありません。**
> Copilot のエージェントがターミナルのコマンドを自動で実行してくれます。
> ただし、最初のセットアップ（Git の設定、GitHub CLI のログインなど）では使います。

## 10. Copilot を開く・初期設定する

### Step 1: GitHub アカウントで VS Code にサインインする

Copilot を使うには、まず VS Code を GitHub アカウントに接続する必要があります。

1. VS Code の **左下** にある **人のアイコン**（アカウントアイコン）をクリック
2. **「Sign in with GitHub to use GitHub Copilot」** をクリック
3. ブラウザが開くので、GitHub にログインして **Authorize** をクリック
4. VS Code に戻ると、サインインが完了している

> サインインが完了すると、左下のアイコンにあなたの GitHub ユーザー名が表示されます。

### Step 2: Copilot Chat を開く

サインインが完了したら、Copilot Chat を開きます。

| 方法 | 手順 |
|------|------|
| **キーボード（最速）** | `Cmd + Shift + I` |
| **サイドバーから** | 左サイドバーの **Copilot アイコン**（吹き出しにキラキラのマーク）をクリック |
| **メニューから** | メニューバーの **View** → **Chat** |

画面の右側（または別パネル）に Copilot Chat が表示されます。

### Step 3: 動作確認する

Chat の入力欄に以下を入力して `Enter` を押してください：

```
こんにちは。あなたは何ができますか？
```

AI が日本語で返答すれば、セットアップ完了です。

### うまくいかない場合

- **「Copilot is not available」と表示される** → GitHub のサインインを再確認。左下のアカウントアイコンから再ログイン
- **Chat パネルが見つからない** → 拡張機能「GitHub Copilot Chat」がインストールされているか確認（[セクション 8](#8-推奨拡張機能のインストール) を参照）
- **応答が極端に遅い** → インターネット接続を確認

## 11. Copilot の使い方（Agent・Edit・モデル選択）

Copilot Chat にはいくつかの「モード」があり、やりたいことに応じて使い分けます。

### Chat パネルの見方

```
┌─────────────────────────────────────┐
│  [Ask ▾]  [モデル名 ▾]             │  ← ①モード選択  ②モデル選択
│─────────────────────────────────────│
│                                     │
│  （会話の履歴がここに表示される）    │
│                                     │
│─────────────────────────────────────│
│  メッセージを入力...          [送信] │  ← ③ここに指示を入力
└─────────────────────────────────────┘
```

### ① モード選択（もっとも重要）

Chat パネル上部にあるドロップダウン（初期値は「Ask」）で切り替えます。

| モード | 何ができる？ | いつ使う？ |
|--------|------------|-----------|
| **Ask** | 質問に答えてくれる（読み取り専用） | コードの意味を知りたい、エラーの原因を聞きたいとき |
| **Edit** | 指定したファイルを直接編集してくれる | 「この関数を修正して」「コメントを追加して」など、部分的な修正 |
| **Agent** | ファイルの作成・編集・ターミナル操作などを自律的に実行 | **このテンプレートの主要モード。計画実行や大きな作業はこれ** |

#### Agent モード（主に使うモード）

**Agent モードが、このテンプレートの中核機能です。**

Agent モードでは、Copilot が：
- ファイルを読み書きする
- ターミナルでコマンドを実行する
- 複数のファイルをまたいだ大きな変更を行う
- Git の操作（コミット・プッシュ・PR 作成）を行う
- 他の AI エージェント（実装担当、テスト担当など）に作業を指示する

**使い方:**
1. モード選択で **Agent** を選ぶ
2. やりたいことを日本語で入力する
3. Copilot が計画を立てて、自動で実行する
4. ファイルの変更提案が表示されたら、**Accept**（承認）または **Discard**（却下）を選ぶ
5. ターミナルでコマンドを実行する場合、**Continue**（実行する）を選ぶ

> **Agent モードでは、Copilot が提案する変更やコマンドに対して**
> **あなたが「OK」を出すまで実行されません。** 安心して使えます。

#### Edit モード

ファイルの部分的な修正に便利です。

**使い方:**
1. エディタで編集したいファイルを開く
2. モード選択で **Edit** を選ぶ
3. 「この関数にコメントを追加して」などと入力
4. 変更の差分（ビフォー・アフター）が表示される
5. 良ければ **Accept**、やり直すなら **Discard**

#### Ask モード

読み取り専用の質問モード。コードを変更しません。

**使い方:**
1. 質問したいコードがあれば、エディタで選択（ドラッグ）する
2. モード選択で **Ask** を選ぶ
3. 「このコードは何をしている？」などと入力
4. 解説が表示される

### ② モデル選択

Chat パネル上部の2つ目のドロップダウンで、AI のモデル（頭脳の種類）を選べます。
モデルは日々更新されるため、以下は 2026年2月時点の代表的なものです。

#### 無料プラン（Copilot Free）で使えるモデル

| モデル | 特徴 | おすすめ場面 |
|--------|------|-------------|
| **GPT-4o mini** | 軽量・高速 | 簡単な質問、コード補完 |
| **Claude Sonnet 4** | コード理解が得意 | コードの解説、中程度の実装 |

> 無料プランではモデルの選択肢や使用回数に制限があります。

#### 有料プラン（Pro / Pro+）で使えるモデル

**プレミアムリクエスト ×1（低コスト・高性能）のモデル:**

| モデル | 特徴 | おすすめ場面 |

| **ChatGPT 5.2** | OpenAI の最新高性能モデル | 一般的な質問、コード修正 |
| **Claude Sonnet 4.5** | コード品質が高く高速 | 日常的なコーディング全般 |
| **Gemini 3.0 Pro** | Google の高性能モデル | 幅広いタスク |

> 上記以外にも ×1 モデルは多数あります。最新のリストは Chat パネルのモデル選択ドロップダウンで確認できます。

**プレミアムリクエスト ×2 以上（超高性能）のモデル:**

| モデル | 特徴 | おすすめ場面 |
|--------|------|-------------|
| **Claude Opus 4.6** | 最も高性能、複雑な推論が得意 | 大規模な実装、オーケストレーション |
| **GPT 5.3 Codex** | OpenAI のコード特化最上位モデル | 大規模なコード生成、複雑なリファクタリング |
| **o3-pro** | 推論特化、高精度 | ロジックが複雑な問題 |

> Pro+ プランでは月あたりのプレミアムリクエスト上限が大きいため、高性能モデルを気軽に使えます。

#### おすすめの使い分け

| 用途 | おすすめモデル |
|------|---------------|
| **ちょっとした質問** | GPT-4o mini / Claude Sonnet 4 |

> Chat パネルのドロップダウンで最新のモデルを確認してください。

### ③ オーケストレーションで使う AI モデルの設定

このテンプレートの AI エージェント（Orchestrator、Implementer 等）は、それぞれ使用するモデルが設定されています。

**デフォルト設定:** すべてのエージェントに **Claude Opus 4.6** が設定されています。

#### モデルを変更したいとき

エージェントの設定ファイルは `agents/` フォルダにあります。
各ファイルの先頭（frontmatter）にある `model:` を書き換えるだけです。

```
# 例: agents/orchestrator.agent.md（または .github/agents/orchestrator.agent.md）の先頭
---
model: Claude Opus 4.6 (copilot)    ← ここを変更する
---
```

**一括変更する場合は、Copilot Chat（Agent モード）で以下のように指示するだけで OK です:**

```
全エージェントの AI モデルを Claude Sonnet 4.5 に変更して
```

#### コスト vs 性能のバランス

| 設定パターン | 説明 |
|-------------|------|
| **全エージェント Opus 4.6** | 最高品質（デフォルト、Pro+ 推奨） |
| **Orchestrator だけ Opus、他は Sonnet** | コスト削減しつつ司令塔の判断精度を確保 |
| **全エージェント Sonnet 4.5** | 低コストで十分な品質（Pro プランに最適） |

> 高性能モデルはプレミアムリクエストを多く消費します。
> 月のリクエスト上限が気になる場合は、Orchestrator 以外を低コストモデルにすると効果的です。

### ③ 日本語での指示のコツ

Copilot は日本語を理解します。以下のポイントを意識すると、より良い結果が得られます。

- **具体的に書く**: ✕「いい感じにして」→ ◯「エラーメッセージを日本語にして」
- **ゴールを伝える**: 「ユーザーがログインできる機能を作りたい」
- **条件を伝える**: 「テストも一緒に作って」「既存のコードは変えないで」

## 12. やりたいことを計画に落とし込む

このテンプレートでは、**まず「やりたいこと」を計画書に書き、計画に沿って AI が自動で実装する** という流れで開発します。

### なぜ計画が必要？

AI に「〇〇を作って」とだけ言うと、方向性がブレたり、抜け漏れが出たりします。
計画書を作ることで：
- AI が「何を作るべきか」を正確に理解できる
- 複数のタスクを順序立てて進められる
- 進捗を追跡できる（何が終わっていて、何が残っているか）

### Step 1: やりたいことを Copilot に伝える

Copilot Chat を **Agent モード** で開き、以下のように伝えます。

```
私は○○をやりたい。これを実現するための計画を作ってほしい。
具体的には、△△と□□の機能が欲しい。
```

**例:**
```
私は家計簿アプリを作りたい。
必要な機能は：
- 収入と支出を記録できる
- 月ごとの合計を表示できる
- カテゴリ別に分類できる
この要件を docs/requirements.md に書き、計画を docs/plan.md に追加してほしい。
```

### Step 2: Copilot が計画を作成する

Copilot は以下のことを自動で行います：
1. `docs/requirements.md` に要件を記載
2. `docs/plan.md` にタスクを追加（どの順番で何を作るか）
3. GitHub Issue を作成（各タスクのチケット）
4. GitHub Projects に追加（進捗ボードで可視化）

### Step 3: 計画を確認して実行する

計画ができたら、以下のコマンドで実行を開始します：

```
計画に従い作業を実施して
```

AI が `docs/plan.md` の **Next**（次にやるタスク）を読み取り、実装を自動で開始します。

### 計画書の構造

```
docs/plan.md
├── ロードマップ     ← プロジェクト全体の大まかな予定
├── Next（1〜3件）   ← 今やるタスク（AI はここだけ実行する）
├── Backlog          ← 将来やるタスク（順番待ち）
└── Done             ← 完了したタスク
```

## 13. 途中で計画を変更・追加する

**開発の途中でも、いつでも計画を変更したり、新しいやりたいことを追加できます。**

### 新しいタスクを追加したいとき

Copilot Chat（Agent モード）で以下のように伝えます：

```
計画を修正して。○○の機能を追加したい。
```

**例:**
```
計画を修正して。家計簿にグラフ表示機能を追加したい。
円グラフでカテゴリ別の支出割合を表示できるようにしたい。
```

Copilot が自動で：
1. `docs/plan.md` にタスクを追加
2. GitHub Issue を作成
3. GitHub Projects に追加

### タスクの優先順位を変えたいとき

```
計画を修正して。グラフ表示機能を最優先にしたい。
現在の Next にあるタスクより先にやりたい。
```

### やらないことにしたいとき

```
計画を修正して。○○の機能はやめたい。理由は△△だから。
```

### 注意点

- **AI が作業中でなければ**、いつでも計画変更できます
- AI が作業中の場合は、その作業が完了してから計画変更してください
- 計画変更トリガーフレーズ：「計画を修正して」「計画を見直して」「新しい要件を追加して」「Backlog に追加して」

## 14. 開発の概念ガイド（Issue・PR・Project）

ソフトウェア開発では、作業を整理・追跡するための仕組みがあります。
ここでは、このテンプレートで使う主要な概念を説明します。

### リポジトリ（Repository）

**コードや書類を保管する「プロジェクトフォルダ」** のようなもの。
GitHub 上にあり、変更の履歴もすべて記録されます。

### ブランチ（Branch）

**「作業用のコピー」** のようなもの。

```
main（本番） ─────────────────────────→
              \                    /
               feat/新機能 ───────  ← 作業用ブランチ
```

- **main ブランチ**: 常に正しい状態を保つ「本番」
- **作業ブランチ**: 新しい機能を作るときに main から分岐させる。完成したら main に合流させる（マージ）

### Issue（イシュー）

**「やることリスト」の1項目**。
「○○の機能を追加する」「△△のバグを直す」のように、作業内容を記録します。

- 番号が振られる（例: #1, #15, #25）
- 完了したらクローズ（閉じる）する
- ラベル（タグ）で分類できる（例: `enhancement`, `bug`）

### プルリクエスト（Pull Request / PR）

**「この変更を本番に入れてもいいですか？」** という提案のこと。

1. 作業ブランチでコードを変更する
2. PR を作成して「こんな変更をしました」と説明を書く
3. 他の人（や AI）がレビューする
4. 問題なければ **マージ**（main に合流）する

PR を通すことで、おかしな変更が本番に混入するのを防ぎます。

### CI（継続的インテグレーション）

**コードを変更するたびに自動でテストを実行する仕組み**。

PR を出すと、GitHub Actions が自動で以下をチェックします：
- コードの書き方ルール（lint）
- 型チェック（mypy）
- テストの実行（pytest）
- ポリシーチェック（禁止操作がないか）

全部パスしないとマージできません。

### GitHub Projects

**「プロジェクトの進捗管理ボード」**。
Issue や PR をカンバンボード（Todo → In Progress → Done）で管理します。

```
Todo          In Progress     Done
┌───────┐    ┌───────┐    ┌───────┐
│ #15   │    │ #20   │    │ #10   │
│ #16   │    │       │    │ #11   │
│ #17   │    │       │    │ #12   │
└───────┘    └───────┘    └───────┘
```

### このテンプレートでの使われ方

```
docs/plan.md   ←→   GitHub Issues   ←→   GitHub Projects
（計画書）          （個別タスク）        （進捗ボード）
```

1. `docs/plan.md` に計画を書く（Next = 今やるタスク、Backlog = 将来やるタスク）
2. 各タスクに対応する GitHub Issue を作成する
3. GitHub Projects で進捗を可視化する
4. タスク完了 → PR マージ → Issue が自動でクローズ → Projects が自動で Done に移動

## 15. プルリクエストの確認とマージ

PR の確認とマージは **VS Code 上で** 行えます（推奨拡張機能をインストール済みの場合）。

### VS Code で PR を確認・マージする方法

**GitHub Pull Requests** 拡張機能を使います。

1. **左サイドバー** に GitHub のアイコン（猫のアイコン付き）が表示される
2. クリックすると、現在のリポジトリの PR 一覧が見える
3. PR をクリックすると：
   - **変更されたファイルの差分**（ビフォー・アフター）が見える
   - **レビューコメント** が見える
   - **CI の結果** が見える
4. 問題なければ、PR の詳細画面で **「Merge Pull Request」** ボタンをクリック

#### 手順の詳細

```
① サイドバーの GitHub アイコンをクリック
② 「Pull Requests」セクションを展開
③ 対象の PR をクリック
④ 「Description」タブで PR の説明を確認
⑤ 「Files Changed」で変更内容を確認
⑥ 「Checks」で CI の結果を確認（全部 ✓ なら OK）
⑦ 問題なければ「Merge Pull Request」をクリック
⑧ マージ方法を選択（通常は「Create a merge commit」で OK）
⑨ 「Confirm Merge」をクリック
```

### GitHub Web で PR を確認・マージする方法（代替手段）

VS Code が使えない場面では、ブラウザからも操作できます。

1. https://github.com にアクセスしてログイン
2. 対象のリポジトリページを開く
3. **「Pull requests」** タブをクリック
4. 対象の PR をクリック
5. **「Files changed」** タブで変更内容を確認
6. **「Checks」** で CI の結果を確認
7. ページ下部の **「Merge pull request」** → **「Confirm merge」** をクリック

### AI エージェントの仕組み

```
あなた（ユーザー）
  │  「計画に従い作業を実施して」

  │  タスクを分解して各担当に指示
```

1. 計画書を読んで次のタスクを把握
2. コードを実装
3. テストを作成
4. 3 方向から監査
5. PR（変更提案）を作成
6. あなたの承認を待ってマージ

## 17. 困ったら Copilot に聞こう

**VS Code の使い方、プログラミングの用語、エラーの意味など、わからないことがあればなんでも Copilot に聞いてください。**

Copilot Chat（`Cmd + Shift + I`）を開いて、日本語で質問するだけで OK です。

### 聞ける内容の例

| こんなとき | こう聞く |
|-----------|---------|
| 用語がわからない | 「プルリクエストって何？」 |
| エラーが出た | 「このエラーの意味と直し方を教えて」 |
| VS Code の操作がわからない | 「ファイルを検索する方法は？」 |
| コードの意味がわからない | （コードを選択して）「これは何をしている？」 |
| やりたいことの進め方がわからない | 「○○の機能を作りたいけど、どう始めればいい？」 |
| Git の操作がわからない | 「git pull って何をするコマンド？」 |

### コツ

- **Ask モード** で聞くと、コードを変更せずに教えてくれる
- 質問は日本語で OK。英語より日本語のほうが意図が正確に伝わることが多い
- エラーメッセージをそのまま貼り付けて「これどういう意味？」と聞くのが最速
- 「初心者にもわかるように説明して」と付け加えると、丁寧に解説してくれる

## 18. トラブルシューティング

### 「Copilot が反応しない」

- Copilot のプランが有効か確認: https://github.com/settings/copilot
- VS Code で GitHub にサインインしているか確認:
  - 左下のアカウントアイコン → 「Sign in to GitHub」

### 「推奨拡張機能の通知が出ない」

- `Cmd + Shift + P` → 「Extensions: Show Recommended Extensions」

### 「git push でエラーが出る」

- `gh auth login` で GitHub にログインし直す
- HTTPS で認証されているか確認: `gh auth status`

### 「Python が見つからない」

```bash
# Homebrew で Python をインストール
brew install python@3.11

# 確認
python3 --version
```

### 「ターミナルに何も表示されない / フリーズしている」

- `Ctrl + C` で現在のコマンドを中断できる
- ターミナルタブの **ゴミ箱アイコン** で閉じて、新しいターミナルを開く

### 「Agent モードで Copilot がファイルを変更しない」

- モードが **Agent** になっているか確認（Ask や Edit では自律操作しない）
- 変更の提案が画面に表示されているのに気づいていない場合がある → **Accept** ボタンを探す
